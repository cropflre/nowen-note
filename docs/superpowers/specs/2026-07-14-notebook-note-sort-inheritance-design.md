# 笔记排序继承修复设计

## 问题

根级笔记本选择“名称升序”等排序方式后，子文件夹会按规则排序，但子文件夹中的笔记仍可能保持手动顺序。当前实现通过默认排序对象的引用相等和模块级可变状态判断是否继承，结果会受到对象创建方式和渲染顺序影响。

## 预期行为

- 根级排序是未单独设置层级的默认排序规则。
- 未单独设置排序的子文件夹，其子文件夹和直属笔记都继承根级排序。
- 明确为某个文件夹设置排序后，该层级继续使用自己的规则，不受根级规则覆盖。
- 手动排序、置顶优先级及现有拖拽限制保持不变。

## 实现方案

移除依赖对象引用和模块级状态的隐式继承，改为显式计算每个父级的有效排序规则：有本层设置时使用本层设置，否则使用根级设置。文件夹树排序与笔记排序共用同一份有效规则，确保两者行为一致。

保持排序函数为纯函数，不在构建文件夹树时写入影响笔记排序的全局状态。仅调整排序规则解析和调用链，不改动界面及持久化数据结构。

## 验证

- 增加回归测试：根级“名称升序”能排序嵌套文件夹中的笔记。
- 增加回归测试：子文件夹明确设置“手动排序”时不继承根级排序。
- 运行相关前端单元测试、类型检查和构建验证。
