# “拆分文档”入口移入更多菜单设计

## 目标

移除编辑器右下角的“拆分文档”悬浮按钮，将相同能力放入桌面端和移动端的“更多”菜单，减少正文区域遮挡，并保持不同终端的操作入口一致。

## 交互设计

- 仅当当前笔记满足现有拆分条件时，更多菜单显示“拆分文档”。
- 菜单项沿用剪刀图标，点击后先关闭当前菜单，再打开现有拆分预览弹窗。
- 不满足拆分条件时不显示菜单项，不新增禁用态或解释文案。
- 删除原有右下角悬浮入口，不保留重复入口。

## 组件边界

- `EditorPaneRuntime` 继续负责计算拆分条件、保存首选标题层级，以及持有拆分弹窗状态。
- `EditorPane` 通过可选属性接收“是否可拆分”和“请求拆分”回调，仅负责在桌面端和移动端更多菜单中展示并触发该操作。
- 现有拆分 API、事务、权限判断、弹窗和完成后的刷新流程保持不变。

## 验证

- 回归测试验证可拆分时桌面端与移动端菜单均提供入口，点击会调用请求拆分回调并关闭菜单。
- 回归测试验证运行时外壳不再渲染悬浮“拆分文档”按钮。
- 运行相关定向测试、TypeScript 生产构建，并在真实页面确认悬浮入口消失且更多菜单入口可打开拆分弹窗。
