# 顶部笔记标签快速切换设计

## 背景

开启顶部笔记标签页后，标签数量增多会产生横向滚动，用户难以快速判断已打开哪些笔记并切换到目标笔记。Issue #285 希望标签溢出时可以折叠显示，或通过竖向列表切换。

## 方案

保留现有单行横向标签栏，在标签栏右侧增加固定的“全部标签”按钮。按钮不参与横向滚动，因此标签较多时始终可见。点击后弹出竖向列表，展示当前全部已打开标签。

列表行为：

- 当前标签使用高亮和选中标记。
- 点击标签切换笔记并关闭列表。
- 每项提供关闭按钮，关闭当前标签时沿用现有相邻标签切换规则。
- 标签数量较多时列表内部滚动，不增加编辑区顶部高度。
- 点击列表外部或按 `Escape` 关闭列表。

## 边界

- 不改变现有标签数量上限、固定标签、拖拽排序和右键菜单行为。
- 不新增后端偏好字段；“全部标签”入口作为标签栏的默认能力直接提供。
- 仅桌面端显示，保持现有移动端标签栏策略不变。

## 测试

- 纯布局逻辑测试：验证下拉列表定位值会被限制在可视区域内。
- 组件测试：验证按钮打开列表、当前项状态、点击切换、关闭标签和 `Escape` 收起。
- 运行前端相关测试与生产构建，确认类型和打包通过。
