# 导入微信收藏

Nowen Note 支持导入 [WeChatDataAnalysis](https://github.com/LifeArchiveProject/WeChatDataAnalysis) 导出的微信收藏 JSON ZIP。

## 导出准备

1. 在 WeChatDataAnalysis 中打开「收藏」。
2. 选择需要导出的收藏类型，或保留“全部”。
3. 导出格式选择 **JSON**。
4. 开启媒体导出，生成包含 JSON 与本地媒体文件的 ZIP。
5. 不要手动解压、重命名或重新压缩包内文件，以免媒体 MD5 和路径关联失效。

Nowen Note 同时兼容：

- `dataset: "favorites"` 的原始收藏 JSON；
- WeChatDataAnalysis 当前版本生成的收藏聊天归档 JSON，其中收藏内容位于 `conversations[].messages`。

## 导入步骤

1. 打开「设置 → 数据管理 → 导入」。
2. 选择「数据迁移 → 微信收藏」。
3. 选择导出的 ZIP。
4. 等待服务端预检完成，检查收藏数量、媒体可用率、类型分布和重复来源数量。
5. 配置：
   - 根笔记本名称；
   - 是否按年份创建子笔记本；
   - 是否保留微信标签；
   - 媒体缺失时是否继续导入正文；
   - 重复数据处理方式；
   - 要导入的收藏类型。
6. 点击「开始导入」。
7. 在结果报告中检查新增、更新、跳过、部分成功、失败和附件数量。

## 重复数据处理

Nowen Note 使用微信收藏的 `serverId`、`localId` 或稳定内容摘要记录来源映射。

- **跳过**：已经导入过的收藏不再创建新笔记，适合日常增量导入。
- **更新**：更新既有笔记的正文、附件、目录、时间和标签。
- **副本**：始终创建一篇新笔记，同时保留原始来源标识供审计。

来源映射按用户和工作区隔离，同一收藏可以分别导入个人空间和不同工作区。

## 内容映射

- 文本和微信笔记转换为可编辑富文本；
- 图片和表情进入附件系统并在正文中显示；
- 视频、语音和普通文件进入附件系统，并以可播放或可打开的链接展示；
- 网页、音乐、商品、视频号和小程序保留安全的 HTTP/HTTPS 链接；
- 位置信息保留名称、地址和经纬度跳转；
- 原来源、收藏类型和收藏时间写入笔记末尾；
- 收藏标签复用或创建为 Nowen Note 标签；
- 默认创建「微信收藏 / 年份」目录结构。

## 安全限制

导入在服务端进行，不会在浏览器中完整解压大型 ZIP。系统会拒绝或跳过：

- ZIP 路径穿越、绝对路径和 Windows 盘符路径；
- 异常高压缩比、条目过多或解压后总体积过大的压缩包；
- 超出附件大小限制的单个媒体文件；
- 可执行文件、脚本、嵌套压缩包和高风险 HTML/SVG 文件；
- 嵌套层级异常的 JSON。

临时上传文件无论成功或失败都会清理，超过 24 小时的遗留临时目录也会定期回收。

## 常见问题

### 预检提示“未识别到微信收藏数据”

确认在 WeChatDataAnalysis 中选择的是「收藏」数据集和 JSON 格式。普通聊天记录 ZIP 不会被当成微信收藏导入。

### 媒体缺失但正文仍成功导入

导出工具可能没有找到微信本地媒体，或者 ZIP 中缺少与 MD5/文件 ID 对应的资源。默认策略会保留正文和媒体占位提示；关闭“媒体缺失时继续导入正文”后，该收藏会标记为失败。

### 再次导入会不会产生重复笔记

默认使用“跳过”策略，不会重复创建。需要同步后来补齐的媒体或内容时，选择“更新”。

### 导入到哪个空间

微信收藏导入使用当前激活的个人空间或工作区。工作区导入要求编辑者以上权限；个人空间导入仍受管理员的个人导入开关控制。
