# 多服务端配置与本地 → NAS 安全迁移

## 功能边界

桌面端的「服务端与迁移中心」包含三类互不等价的操作：

- **切换服务器**：只改变当前客户端连接的实例与账号，不复制数据。
- **迁移数据**：把一个实例的个人空间一次性复制到另一个实例。
- **导入备份**：从用户选择的备份文件恢复数据，继续使用数据管理模块完成。

迁移不是双向同步。完成本地 → NAS 迁移后，推荐让电脑、手机和平板统一连接 NAS 实例；本地实例只保留为只读备份或应急恢复源。

## 多服务端配置

每条配置包含：

- 自定义名称；
- 服务端 URL；
- 类型（本地、NAS、远程、演示）；
- 当前账号；
- 连接状态；
- 最近检测和最近使用时间；
- 登录令牌。

密码只用于保存配置时登录一次，不写入配置。切换前会显示目标服务器、账号和未同步队列风险，并要求明确确认。

## 迁移协议 v2

接口前缀：

```text
/api/user-migration/v2
```

安全流程：

1. `GET /preflight` 扫描笔记本、笔记、标签、历史版本、任务和附件。
2. `POST /target-preview` 检查历史迁移记录和源端变化。
3. `POST /source-backup` 在源实例的数据目录创建 ZIP 安全备份。
4. `GET /export` 取得带快照哈希的数据集。
5. `POST /import` 事务导入元数据并返回 ID 映射。
6. 逐个下载和上传附件；两端都校验大小与 SHA-256。
7. `POST /rewrite` 重写笔记和任务中的附件引用。
8. `POST /complete` 在目标端重新读取所有附件，复核数量、大小和 SHA-256。
9. 成功后切换客户端到 NAS。

## 冲突策略

- **跳过已有**：已迁移项目保留目标版本，只补充新内容。
- **更新迁移副本**：只更新历史迁移创建且仍可追踪的副本，不覆盖目标端原生内容。
- **保留两份**：源内容变化时新建带“迁移副本”后缀的内容。

服务端通过以下组合追踪源项目：

```text
targetUserId + sourceInstanceId + sourceUserId + entityType + sourceId
```

因此重复执行同一快照会复用目标 ID，不会静默生成重复副本。

## 中断、续传和撤销

客户端把迁移 ID、源快照、ID 映射和已完成附件记录写入本地 checkpoint。中断后使用相同源、目标和策略再次执行时：

- 元数据导入按迁移台账幂等复用；
- 已完成且哈希一致的附件跳过传输；
- 未完成附件从断点继续；
- 最后仍执行目标端完整校验。

失败后可以调用 `POST /rollback`。回滚只删除该迁移 ID 创建并可追踪的目标副本，不影响源端数据、源端 ZIP 备份或目标端原生内容。

## 备份位置

默认目录：

```text
<ELECTRON_USER_DATA>/migration-backups/
```

非 Electron 部署回退到：

```text
<cwd>/data/migration-backups/
```

备份包含 `manifest.json`、`data.json` 和笔记/任务附件二进制。源端存在物理文件缺失时，自动备份和迁移都会被阻止，避免生成看似成功但实际残缺的数据副本。
